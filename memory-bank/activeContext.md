## Текущий активный контекст

### Итерация 1 (исторический срез)

На данном этапе в проекте:

- Создан базовый макет дашборда с тремя колонками.
- Левая панель (260px, серая) показывает библиотеку блоков: Hero, Текст+Изображение, Карточки товаров, Форма, Отзывы, FAQ, Footer.
- Центральная область — канвас с светло-серым фоном и мелкой сеткой, поддерживает drag & drop блоков с помощью **dnd-kit**.
- Блоки на канвасе представлены компонентом `BlockOnCanvas`, они сортируемые (можно менять порядок перетаскиванием).
- Есть состояние для списка блоков на канвасе, включая сохранение их типа и настроек.
- Реализован инспектор свойств в правой панели (280px, серая): при клике по блоку он подсвечивается, а справа отображаются поля настройки (разные для Hero, Карточек товаров, Текст+Изображение, Формы, Отзывов, FAQ и Footer).
- Используются компоненты **shadcn/ui** (`Input`, `Checkbox`, `RadioGroup`, `Textarea`, `Label`) для формы инспектора.

**Следующий шаг (на тот момент)**: донастроить/расширить библиотеку UI-компонентов shadcn (если потребуется), продумать сохранение/загрузку схемы ТЗ (интеграция с Supabase).

### Итерация 2 (текущий активный контекст)

На текущем этапе:

- Редактор мигрирован на **Puck**; используется конфигурация блоков и механизм нормализации/миграции данных.
- Генерация ТЗ вынесена в backend (`/api/generate-spec`) и работает в потоковом режиме (SSE).
- AI-функция интегрирована (SDK `openai` + DeepSeek endpoint/model) с батчевой обработкой блоков и retry-логикой.
- Добавлены квоты на генерацию (`/api/generate-spec/quota`) и защита от превышения лимита.
- Проекты сохраняются в Supabase: список, создание/обновление, загрузка, удаление.
- Сгенерированное ТЗ (`spec`) сохраняется отдельно от структуры блоков.
- Работают auth/middleware для Supabase и пользовательский контекст.
